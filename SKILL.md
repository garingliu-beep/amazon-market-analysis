## 执行规则（强制）

### 规则一：任务验证

**每次完成一个Step后，必须进行验证**，确保输出准确、完整：

1. **数据验证**：检查获取的数据是否合理（如价格范围、Review数量级）
2. **逻辑验证**：检查计算结果是否符合常识（如利润率不能为负数或超过100%）
3. **完整性验证**：检查该Step的所有要求项是否都已覆盖
4. **向用户确认**：关键Step完成后，向用户简要汇报结果，确认方向正确后再继续

**验证失败处理**：
- 数据异常 → 回到数据获取步骤重新获取
- 逻辑错误 → 检查计算公式和参数
- 缺失项 → 补充完整后再进入下一步

### 规则二：禁止跳步

**严格按照流程顺序执行，不允许跳过任何Step**：

1. 必须完成「数据确认」后才能进入正式调研流程
2. 必须完成Step 1才能进入Step 2，依此类推
3. 每个Step的输出是下一个Step的输入，不得省略
4. 如遇阻塞，必须向用户说明阻塞原因和解决方案，不得自行跳过

### 规则三：数据确认（强制首步）

**在开始任何调研之前，必须先执行数据确认流程**：

1. 向用户询问：目前手上有哪些相关数据？
2. 等待用户明确回复后，再判断使用哪种模式
3. 确认数据清单后，向用户复述确认，得到用户认可后再开始
4. **禁止在未确认数据的情况下直接开始调研**

### 规则四：亚马逊数据抓取必须优先使用模拟浏览器（强制）

**亚马逊前台数据（搜索结果、产品页、Best Sellers等）必须优先使用 `agent-browser` 模拟浏览器方式抓取，禁止直接使用 `web_fetch` 抓取亚马逊页面。**

**原因**：亚马逊对 `web_fetch` 等直接HTTP请求有严格的反爬机制（返回503/CAPTCHA），而 `agent-browser` 通过模拟真实浏览器行为可有效规避。

**执行要求**：

1. **亚马逊前台数据**：必须使用 `agent-browser` 命令链
   ```bash
   agent-browser open "https://www.amazon.com/s?k=关键词"
   agent-browser wait --load networkidle
   agent-browser snapshot -i
   ```
2. **多页数据**：逐页抓取，每页之间等待页面加载完成
   ```bash
   agent-browser open "https://www.amazon.com/s?k=关键词&page=2"
   agent-browser wait --load networkidle
   agent-browser snapshot -i
   ```
3. **产品详情页**：同样使用 `agent-browser` 抓取
   ```bash
   agent-browser open "https://www.amazon.com/dp/ASIN"
   agent-browser wait --load networkidle
   agent-browser snapshot -i
   ```
4. **会话管理**：抓取完成后必须关闭浏览器会话
   ```bash
   agent-browser close
   ```
5. **拦截处理**：如果 `agent-browser` 也返回 "Something went wrong" 错误：
   - 等待 5-10 秒后重试
   - 尝试先打开 amazon.com 首页，再导航到目标页
   - 如果持续失败，走降级策略

**禁止行为**：
- ❌ 直接使用 `web_fetch` 抓取 `amazon.com` 链接
- ❌ 使用 `curl` 等命令行工具直接请求亚马逊

**允许行为**：
- ✅ 使用 `web_fetch` 抓取非亚马逊网站（1688、Google Trends、第三方分析等）
- ✅ 使用 `mimo_web_search` 搜索亚马逊相关二手数据

### 规则五：AI 边界（强制遵守）

**明确区分「规则计算」和「AI 推断」，不得混淆**：

| 类别 | 负责方 | 说明 | 报告标注 |
|------|--------|------|----------|
| **硬指标计算** | 规则引擎 | 销售额、销量、PPC、CVR、ACOS、HHI、集中度、供需指数、利润率 | 直接输出，无需标注 |

## 选品决策框架（媛媛方法论）

### 核心原则

**选品的本质**：不是找没人卖的东西，而是找到"有人要、竞争没锁死、现有产品有缺口、自己改得动、算账能赚钱"的细分市场。

### 前置风险排除（必须先做）

#### 1. 侵权风险排查
- **外观专利**：检查产品外观是否涉及已知设计专利
- **发明专利**：检查产品功能是否涉及发明专利
- **品牌商标**：检查品牌名称是否已注册
- **版权**：检查图片、文案是否有版权风险
- **排查工具**：USPTO专利检索、Google Patents、亚马逊品牌注册

#### 2. 认证合规检查
- **FDA**：食品接触、化妆品类
- **FCC**：电子产品、无线设备
- **CPSC**：儿童产品、消费品安全
- **UL/ETL**：电气安全
- **Prop 65**：加州化学物质警告
- **CE**：欧洲市场
- **WEEE/RoHS**：电子废弃物

> ⚠️ **红线**：涉及侵权或认证不合规的产品，利润再高也不做。

### 避坑信号（快速排除不值得做的市场）

在深入分析之前，先用以下信号快速排除明显不值得做的市场：

#### 🚫 信号一：头部锁死
- 前 10% 链接或品牌吃掉大部分销量（Top10集中度 > 70%）
- 新品很难进入前 20%（新品存活率 < 5%）
- 头部价格持续下压（近6个月价格下降 > 15%）
- **判断**：直接排除，不要硬打

#### 🚫 信号二：广告吞利润
- PPC 高（核心词 CPC > $2.5）
- 转化率低（品类平均 CVR < 8%）
- 售价低（主流价格带 < $20）
- ACOS 压缩毛利空间（ACOS > 毛利率的60%）
- **判断**：广告费吃掉全部利润，不值得做

#### 🚫 信号三：产品不可分层
- 有效决策维度少（买家只看价格和评分，无其他区分维度）
- 特征高度同质化（所有产品形态几乎一样）
- 组合样本不足或没有产品意义
- **判断**：没有差异化空间，只能打价格战

> ⚠️ 命中任一避坑信号，应直接建议用户放弃或转向，不进入后续详细分析。

### 核心考量因素（按优先级排序）

#### 1. 客单价（最重要）

| 客单价区间 | 判断 | 说明 |
|-----------|------|------|
| < $15 | ❌ 不建议 | 利润空间极小，推广几乎无空间 |
| $15-30 | ⚠️ 特别慎重 | 每个环节都要充分测算，稍有偏差就亏损 |
| $30-50 | ✅ 推荐 | 有足够利润空间覆盖推广成本 |
| $50-100 | ✅ 强烈推荐 | 利润空间大，可承受更高推广成本 |
| > $100 | ⚠️ 慎重 | 需要更多信任建设，Review门槛高 |

**$30以下产品特别注意**：
- FBA费用占比高（可能占售价30%+）
- 广告费占比高（CPC相对售价比例高）
- 退货损耗影响大
- 每个环节都要精算，没有容错空间

#### 2. 竞品毛利水平

**分析方法**：
1. 采集TOP 20竞品的售价
2. 估算竞品的采购成本（1688/行业经验）
3. 计算竞品的毛利区间
4. 判断：如果竞品毛利<20%，说明品类利润空间被压缩

**健康毛利标准**：
- 广告前毛利率 ≥ 40% → 优秀
- 广告前毛利率 30-40% → 合格
- 广告前毛利率 < 30% → 高风险

#### 3. 广告压力公式

**核心公式**：
```
ACOS = PPC ÷ (CVR × 平均售价)
```

**判断逻辑**：只看 CPC 高低会误判。必须三个变量一起看：
- PPC 高 → 拿流量贵
- 转化率低 → 流量更难成交
- 售价低 → 广告容错空间更小

**广告费占售价比例红线**：

| 售价区间 | 合理ACoS | 广告费占比红线 |
|---------|---------|--------------|
| $15-30 | 15-20% | ≤ 15% |
| $30-50 | 15-25% | ≤ 15% |
| $50-100 | 20-30% | ≤ 20% |
| > $100 | 25-35% | ≤ 20% |

> 广告费占售价超过 20%，基本没有利润。

#### 4. 可进入空间公式

**核心公式（乘法模型）**：
```
可进入空间 = 市场容量因子 × 非垄断空间因子 × 可运营性因子 × 利润余量因子
```

各因子说明：
- **市场容量因子**：年销售额、年销量、趋势方向（取值0-1）
- **非垄断空间因子**：1 - 前10%链接/品牌/卖家占比（取值0-1）
- **可运营性因子**：广告承压力、评价门槛可追性、新品成功率（取值0-1）
- **利润余量因子**：基准情景净利率（取值0-1）

**判断标准**：
- 可进入空间 > 0.3 → 值得进入
- 可进入空间 0.15-0.3 → 有条件进入
- 可进入空间 < 0.15 → 不建议

> ⚠️ 乘法模型的特性：任一因子极差，整体结果都会被拉低。这比加权打分更能暴露致命短板。

#### 5. 市场体量

| 指标 | 最低门槛 | 理想值 | 说明 |
|------|---------|--------|------|
| 品类月GMV | $100K | $500K+ | 太小没空间 |
| TOP 20月销中位数 | 200件 | 500件+ | 太小不值得做 |
| 关键词月搜索量 | 3000 | 10000+ | 太小流量不够 |

#### 6. 市场竞争度

| 指标 | 友好 | 中等 | 困难 |
|------|------|------|------|
| 首页平均Review | < 300 | 300-800 | > 800 |
| 品牌集中度HHI | < 1500 | 1500-2500 | > 2500 |
| 新品存活率 | > 20% | 10-20% | < 10% |
| 核心词CPC | < $1.0 | $1.0-2.0 | > $2.0 |

#### 7. 预估单量和盈利周期

**预估日单量公式**：
```
预估日单量 = 目标BSR对应的日销量 × 市场份额目标
预估月单量 = 预估日单量 × 30
```

**盈利周期测算**：
```
盈亏平衡月销量 = 固定月成本 / 单件毛利
预计盈利月数 = 盈亏平衡月销量 / 预估月单量
```

**健康标准**：
- 盈利周期 < 3个月 → 优秀
- 盈利周期 3-6个月 → 合格
- 盈利周期 > 6个月 → 慎重

### 差异化策略

#### 标准化产品差异化（不一定需要产品升级）

**SKU配置差异化**：
- **尺寸差异化**：主推非主流尺寸（如大号或小号）
- **数量差异化**：多件套装或单件精品
- **颜色差异化**：非主流颜色（如黑色、粉色）
- **组合差异化**：功能组合套装（如刷子+毛巾+沐浴露）
- **包装差异化**：礼盒装、便携装

**切入点差异化**：
- **人群切入点**：针对特定人群（如老年宠物、幼宠、大型犬）
- **场景切入点**：针对特定场景（如户外、旅行、医疗护理）
- **价格切入点**：高端或低端细分市场

#### 产品设计差异化（更佳）

- **功能创新**：解决用户痛点的新功能
- **材质升级**：更安全、更耐用的材质
- **人机工程**：更易用、更舒适的设计
- **美学设计**：更符合目标市场审美

### 成熟产品 vs 新产品分析

#### 成熟产品分析重点

**评分分析**：
- 类目平均评分是多少？（通常4.2-4.5）
- 评分低于均值的产品有什么问题？
- 评分红线：低于4.0的产品问题严重

**好评分析**：
- 用户最认可的3个点是什么？
- 这些点是否可以复制或超越？

**差评分析**：
- 用户最抱怨的3个点是什么？
- 这些痛点是否可以解决？
- 差评红线：涉及安全、质量的核心问题

#### 新产品分析重点

**消费需求洞察**：
- 用户在解决什么焦虑/需求？
- 现有产品是否满足了这个需求？
- 需求是否真实且持续？

**卖点设计**：
- 核心卖点是否能一句话说清？
- 卖点是否能打动目标用户？
- 卖点是否有差异化？

**市场教育**：
- 用户是否知道这类产品存在？
- 需要多少推广成本让用户认知？
- 市场教育周期有多长？

### 老板仪表盘（8 个核心指标）

老板在做进入决策时，不需要看所有数据，只需看以下 8 个指标：

| 指标 | 说明 | 健康标准 |
|------|------|----------|
| 年销售额 | 市场规模 | > $6M（月GMV > $500K） |
| 年销量 | 需求量级 | 与品类特性匹配 |
| 平均售价 | 利润空间基础 | > $30 |
| PPC | 流量成本 | 结合CVR和售价看 |
| 转化率 | 流量效率 | > 10% |
| ACOS | 广告压力 | < 毛利率的50% |
| 前10%集中度 | 垄断程度 | < 60% |
| 新品推新成功率 | 进入窗口 | > 15% |
| 供需指数>1的机会数 | 可切入方向 | ≥ 3个 |

> 详见 `references/standards.md` 老板仪表盘评分标准。

| 维度 | 权重 | 评分标准 |

| 属性与销量关系 | mimo_web_search + 数据分析 | 搜索品类属性分析报告，或基于用户提供的数据交叉分析 |

### 降级策略（当 agent-browser 抓取失败时）

1. **首选**：`agent-browser` 模拟浏览器抓取（必须首先尝试）
2. **重试**：如果返回错误，等待5-10秒后重试；或先打开首页再导航到目标页
3. **降级1**：`mimo_web_search` 搜索 "amazon [关键词] best sellers top products"，从搜索结果中获取二手数据
4. **降级2**：`mimo_web_search` 搜索 "[关键词] amazon market analysis [年份]"，参考第三方分析报告
5. **降级3**：引导用户提供截图或手动数据

> ❌ **禁止**：直接使用 `web_fetch` 抓取亚马逊页面作为首选方案

> ⚠️ 所有从外部获取的数据都应标注来源，让用户知道哪些是实时数据、哪些是二手信息。

## 入口方式（Entry Point）

本SKILL支持三种入口方式，用户可根据手头信息选择：

### 入口A：消费场景入口（官方推荐）

**适用场景**：用户有一个消费场景方向，想挖掘该场景下的产品机会。

**来源**：亚马逊官方「需求挖金」方法论——从消费场景出发挖掘潜在需求。

**输入**：消费场景描述 + 目标站点

**典型场景**：
- 居家办公（Work from Home）
- 户外运动（Outdoor Sports）
- 节日场景（Holiday/Seasonal）
- 宠物护理（Pet Care）
- 婴儿育儿（Baby Care）
- 厨房烹饪（Kitchen Cooking）
- 卫生间改造（Bathroom Renovation）

**流程**：消费场景 → 关联产品品类 → 品类界定 → 市场分析 → 竞争分析 → 机会评估

**场景→产品映射示例**：
| 消费场景 | 关联产品品类 |
|---------|-------------|
| 户外烧烤 | 烧烤签、烤肉夹、烧烤刷、户外桌椅 |
| 居家办公 | 显示器支架、键盘托、收纳盒、台灯 |
| 婴儿洗澡 | 婴儿浴盆、水温计、淋浴软管、浴巾 |

### 入口B：关键词入口

**适用场景**：用户有一个品类方向，想了解这个市场能不能做。

**输入**：品类核心关键词（英文）+ 目标站点

**流程**：关键词 → 品类界定 → 市场分析 → 竞争分析 → 机会评估

### 入口C：ASIN入口

**适用场景**：用户看到一个竞品或有目标产品，想了解这个产品所在的市场情况。

**输入**：一个或多个ASIN + 目标站点

**流程**：ASIN → 自动识别品类 → 自动筛选竞品池 → 市场分析 → 竞争分析 → 机会评估

**ASIN入口优势**：
- 直接定位到具体细分市场，避免关键词歧义
- 自动建立竞品池（基于BSR、关键词重叠、流量关系）
- 可直接分析目标产品的流量来源和竞争关系

**ASIN入口执行流程**：

1. **产品识别**：通过ASIN获取产品信息（标题、价格、类目、BSR、Review等）
2. **赛道锁定**：自动识别产品所属的细分市场（Niche）和类目层级
3. **竞品池建立**：
   - 从BSR榜单筛选同层级竞品
   - 从搜索结果筛选关键词重叠度高的竞品
   - 从关联流量筛选流量关系紧密的竞品
   - 区分：头部品牌、成长型品牌、潜在竞争对手
4. **流量词反查**：分析目标ASIN的核心成交词、长尾词、场景词
5. **进入后续分析流程**（按模式一/二/三执行）

> 入口方式不改变调研模式（探索级/验证级/决策级），只改变起点和竞品筛选方式。

## 模式一：探索级调研

### 适用场景
用户仅提供关键词或ASIN，需要AI从零开始完成市场摸底。

### 输入要求

**消费场景入口（官方推荐）：**
- 消费场景描述（如"户外烧烤"、"居家办公"、"婴儿洗澡"）
- 目标市场站点（默认US）

**关键词入口：**
- 品类核心关键词（英文）
- 目标市场站点（默认US）

**ASIN入口：**
- 目标ASIN（1个或多个）
- 目标市场站点（默认US）

**选填（有则更精准）：**
- 价格区间偏好
- 是否有供应链方向（如1688关键词）
- 预算范围
- 特别关注的维度
- **多站点验证（推荐）**：是否需要同时分析其他站点（如UK/DE/JP）的数据，交叉验证降低单站点数据风险

### 执行流程

#### Step 1：品类范围界定 + 关键词细分 + 买家决策维度发现

**关键词入口 - 关键词细分方法：**

不要直接盯大词（如bottle、organizer、lamp），用组合公式细分市场：

**关键词组合公式**：属性词 + 场景/人群词 + 产品词

示例：
- lunch box **for women**（人群）
- **under sink** organizer（场景）
- **quiet** fan（属性）
- **stainless steel** water bottle **for gym**（材质+场景）

**关键词筛选标准**：
- 月搜索量：3000-25000（太小没市场，太大竞争激烈）
- 近两个月热度：没有明显下滑
- CPC：尽量低于$1.2（广告成本可控）
- 搜索结果一致性：搜什么出来的就是什么

**买家决策维度发现（新增 — 选品核心引擎）**：

从消费者语言里找决策维度，这是选品的基础。关键词反查不是为了堆词，而是为了理解买家用哪些属性做选择。

**维度发现 6 步流程**：

| 步骤 | 动作 | 说明 |
|------|------|------|
| 1. 核心词 | 收集流量前90%的关键词 | 从搜索结果和工具获取 |
| 2. 词根 | 去主词和泛词 | 去掉"amazon""best""for"等无意义词 |
| 3. AI聚类 | 将词根归成维度 | 按语义相似性分组（如"stainless steel""plastic""silicone"归为"材质"维度） |
| 4. 五维评分 | 对每个维度打分 | 用公式量化维度的业务价值（见下方） |
| 5. AI补全 | 补常识维度 | AI根据品类常识补充可能遗漏的维度 |
| 6. 人工确认 | 用户选3-6个 | 最终维度由用户确认 |

**维度评分公式**：
```
维度得分 = 搜索需求×35% + 区分产品×20% + 购买影响×20% + 可打标×15% + 业务可用×10%
```

各因子说明（1-10分）：
- **搜索需求（35%）**：买家是否真的在搜这个维度的词？搜索量越大，需求越真实
- **区分产品（20%）**：这个维度能否分出不同的产品层？如果所有产品在这个维度上都一样，区分度为0
- **购买影响（20%）**：这个维度是否影响买家的购买决策？材质、尺寸通常是强影响，颜色可能是弱影响
- **可打标（15%）**：能否对每个产品标注这个维度的值？如果无法从标题/描述中识别，打标难度高
- **业务可用（10%）**：这个维度能否指导选品和开发？能否形成产品方向和卖点？

**维度筛选标准**：
- 得分 ≥ 7.0 → 强维度，必须纳入分析
- 得分 5.0-6.9 → 弱维度，可选纳入
- 得分 < 5.0 → 无效维度，排除

> 详见 `references/standards.md` 维度评分标准。

**需求属性分类：**

识别产品属于哪类需求（来自亚马逊官方方法论）：

| 需求类型 | 说明 | 选品策略 | 示例 |
|---------|------|---------|------|
| 基础功能 | 产品的核心功能需求 | 做好基本功能，性价比优先 | 普通淋浴软管 |
| 细分场景 | 特定使用场景的需求 | 场景化定位，差异化卖点 | 露营用长款烧烤签 |
| 配件替换 | 耗材和配件的复购需求 | 复购率高，可做订阅 | 竹签（一次性）、密封圈 |
| 品质升级 | 用户对更好品质的需求 | 品质溢价，品牌定位 | 德国制造不锈钢软管 |

**ASIN入口**：
1. 通过ASIN获取产品信息（标题、价格、类目节点、BSR排名）
2. 自动识别产品所属的细分市场（Niche）
3. 确认类目层级和产品定位
4. 向用户汇报识别结果，确认后继续

**✅ 验证点**：关键词是否符合组合公式，搜索量/CPC是否在合理范围，品类定义是否清晰，维度评分是否≥7.0，需求属性分类是否明确。

#### Step 2A：市场基本面扫描

通过工具获取数据（详见「工具使用指引」），收集以下信息：

**趋势判断：**
- 使用 mimo_web_search 搜索该关键词的Google Trends数据
- 判断趋势形态：上升/稳定/下降/波动
- 识别季节性特征
- 如品类有明显季节性，标注旺季和淡季的月份

**官方消费趋势参考：**

参考亚马逊《全球全品类20大消费趋势报告》（2026年版），判断品类是否契合当前趋势：

| 趋势方向 | 说明 | 关联品类示例 |
|---------|------|-------------|
| AI智感空间 | AI赋能的智能家居产品 | 智能花洒、智能灯具 |
| 健康美妆 | Clean Beauty、天然成分 | 天然洗护、有机护肤 |
| 户外运动 | 户外生活方式产品 | 烧烤工具、露营装备 |
| 可持续生活 | 环保、可重复使用产品 | 不锈钢吸管、竹制餐具 |
| 居家办公 | 远程办公场景产品 | 显示器支架、收纳盒 |

**季节性判断方法：**

1. **BSR历史曲线分析**：查看头部竞品BSR历史曲线（建议2年数据）
   - 头部竞品BSR在每年某月明显上升 = 旺季开始
   - 旺季开始前6个月是入场最佳时机
   - BSR波动：旺季BSR<5000，淡季BSR>50000 = 季节性明显

2. **搜索量季节性分析**：查看关键词12-24个月搜索量变化

3. **双线并行策略**（适用于季节性品类）：
   - 旺季线：季节爆款冲现金流
   - 淡季线：全年中长尾稳定品积累排名

**价格分布：**
- 记录搜索结果中所有可见产品的价格
- 按$5区间分段统计产品数量
- 标注最集中的价格区间（主流价格带）

**Review壁垒：**
- 记录首页自然排名产品的Review数量和评分
- 计算首页平均Review数量
- 判断壁垒等级（详见 `references/standards.md`）

**推广成本评估矩阵：**

| 指标 | 低竞争（绿灯） | 中等竞争（黄灯） | 高竞争（红灯） |
|------|--------------|----------------|--------------|
| 核心词CPC | < $1.0 | $1.0-$2.0 | > $2.0 |
| 首页平均评论数 | < 200 | 200-500 | > 500 |
| 首页平均评分 | < 4.3 | 4.3-4.5 | > 4.5 |
| 品牌集中度HHI | < 1500 | 1500-2500 | > 2500 |

**品牌集中度：**
- 统计首页产品涉及的品牌数量
- 判断是否有单一品牌多ASIN占据多个位置

**产品形态：**
- 识别搜索结果中的主要产品类型/形态
- 标注主流材质、尺寸、功能特征

**五大榜单分析：**

| 榜单 | 分析要点 | 价值 |
|------|---------|------|
| **畅销榜（Best Sellers）** | 当前热销产品，验证需求真实性 | 确认市场有真实需求 |
| **新品榜（New Releases）** | 近期上架的热门新品 | 发现新品机会和趋势 |
| **飙升榜（Movers & Shakers）** | 排名快速上升的产品 | 捕捉快速增长的品类 |
| **心愿单（Wish List）** | 用户想买但未购买的产品 | 发现潜在需求 |
| **礼物榜（Gift Ideas）** | 被作为礼物购买的产品 | 发现节日送礼场景 |

**✅ 验证点**：数据来源标注是否完整，价格范围是否合理，趋势判断是否有数据支撑，五大榜单分析是否覆盖。

#### Step 2B：需求验证

验证需求是否真实稳定：

1. **销量中位数**：计算前20-30个链接的月销量中位数
   - 中位数≥200 → 需求充足
   - 中位数<200 → 需求偏小，谨慎进入

2. **趋势判断**：
   - 平稳或缓慢上涨 → 稳定需求（推荐）
   - 暴涨后快速回落 → 短期热点（新手慎重）

3. **市场vs个体判断**：
   - 多个卖家稳定出单 → 市场大
   - 只有1-2个大链接，其余很弱 → 单个卖家强，不是市场大

4. **需求类型筛选**：
   - 优先：刚需、功能性、更新换代慢的产品
   - 谨慎：时尚性、季节性、跟风热点产品

**✅ 验证点**：需求验证四项是否都已检查，销量中位数是否达到阈值，趋势是否稳定。

#### Step 3：需求与痛点初探 + 用户画像分层

**用户画像分层（基于产品形态的初步推测）：**

> ⚠️ 模式一下没有评论数据，用户画像基于产品形态和使用场景推测，精度较低（★★☆☆☆）。建议在模式二/三中基于评论数据做精确分析。

**需求与痛点提取：**
- 分析TOP产品的标题和五点描述，提取核心卖点
- 识别产品的主要使用场景
- 如能看到部分差评标题/内容，做初步痛点提取
- 使用 mimo_web_search 搜索 "[关键词] reddit review" 或 "[关键词] common complaints"

**✅ 验证点**：人群分类是否合理，痛点提取是否有实际数据支撑。

#### Step 4：利润粗估

- 使用 mimo_web_search 搜索采购成本
- 根据产品大致重量尺寸，估算FBA配送费
- **实时查询佣金率**：mimo_web_search 搜索 "amazon [品类] referral fee percentage"
- 给出粗略的利润区间估算（标注精度较低）

**广告压力初估**：
```
ACOS = PPC ÷ (CVR × 平均售价)
广告费占比 = ACOS × (1 - 退货率)
```
- 判断广告费是否能被利润覆盖

**✅ 验证点**：利润率区间是否在合理范围，成本项是否齐全，广告压力是否可承受。

#### Step 5：风险初筛 + 避坑信号检查

- 检查是否涉及FDA/FCC/CPSC等常见认证要求
- 基于品类特征提示可能的专利风险类型
- **执行避坑信号检查**：头部锁死？广告吞利润？产品不可分层？

**✅ 验证点**：风险项是否覆盖主要类别，避坑信号是否已检查。

#### Step 6：简化属性分析 + 供需指数初算

基于搜索结果中的产品信息，做初步的属性分析：

1. **属性识别**：从产品标题、描述中提取主要属性维度（材质、尺寸、颜色等）
2. **属性分布**：统计各属性值的产品数量分布
3. **初步判断**：哪种属性组合最常见（主流配置）

**特征归一化规范（新增）**：

> ⚠️ 特征归一化决定机会判断会不会跑偏。必须遵守以下规范：

**正确合并**：
- 只合并消费者理解上等价的表达
- 流程：原始识别值 → 标准词义 → 上位特征组
- 保留主流特征和长尾特征的区分

**错误合并（禁止）**：
- 低占比就强行塞进主流类
- 把不同决策含义的特征混在一起（如"防水"和"耐用"是不同决策维度）
- 为了凑样本牺牲产品意义

**供需指数初算**：

对每个主要属性组合，计算供需指数：
```
该组合Listing销量占比 = 该组合的总销量 / 全部样本总销量
该组合Listing数量占比 = 该组合的产品数 / 全部样本产品数
供需指数 = 销量占比 ÷ 数量占比
```

**判断标准**：
- 供需指数 > 1 → 需求强于供给，是机会信号
- 供需指数 = 1 → 供需基本匹配
- 供需指数 < 1 → 供给相对多于需求

> ⚠️ 模式一下供需指数基于搜索结果推测，精度较低。正式机会判定需在模式二/三中基于完整数据验证。

**✅ 验证点**：属性识别是否基于实际产品信息，归一化是否遵守规范，供需指数计算是否正确。

#### Step 7：综合评分

各维度评分（1-10分），评分标准详见 `references/standards.md`：

| 维度 | 权重 | 评分依据 |
|------|------|---------|
| 市场需求 | 30% | 搜索趋势、销量中位数、季节性 |
| 竞争可行性 | 30% | Review壁垒、品牌集中度、推广成本矩阵 |
| 利润空间 | 20% | 利润粗估结果、广告压力 |
| 差异化空间 | 10% | 产品形态差异、供需指数>1的组合数量（初步推测） |
| 风险可控度 | 10% | 合规、专利、供应链（初步筛查）、避坑信号 |

**综合得分 = Σ（各维度得分 × 权重）**

**可进入空间（并行计算）**：
```
可进入空间 = 市场容量因子 × 非垄断空间因子 × 可运营性因子 × 利润余量因子
```
各因子取值0-1，详见 `references/standards.md`。

结论判断：
- ≥ 8.0：强烈建议深入调研（进入模式二/三）
- 6.5-7.9：建议深入调研
- 5.0-6.4：谨慎考虑，需补充数据验证
- < 5.0：不建议

**✅ 验证点**：各维度评分是否有依据，加权计算是否正确，可进入空间是否与综合评分一致。

### 输出

使用报告模板（见 `references/report-templates.md` 模式一部分），数据置信度标注为 ★★☆☆☆。

**HTML报告（必须）**：生成HTML格式报告，保存到 `调研报告/[品类关键词]-mode1-[日期].html`。

## 模式二：验证级调研

### 适用场景
用户已有部分数据，需要AI补充缺失数据并进行交叉验证分析。

### 输入要求

**必填（至少包含其中一类）：**

**类型A - 市场数据：**
TOP产品列表，包含以下字段中的大部分：
- ASIN、品牌、售价、月销量、月销额、Review数量、评分、上架时间、BSR排名、FBA/FBM、变体数
- 来源通常为：卖家精灵、Helium 10、Jungle Scout等工具导出

**类型B - 成本数据：**
- 采购单价、产品重量、产品尺寸、头程运费方式及单价

**类型C - 关键词数据：**
- 核心关键词及其月搜索量、主要长尾词、CPC出价

**选填：**
- 目标售价区间、已知竞品ASIN、自身供应链优势、Review原始数据、Google Trends数据
- **多站点验证（推荐）**：是否需要同时分析其他站点（如UK/DE/JP）的数据，交叉验证降低单站点数据风险

### 执行流程

#### Step 1：数据完整性检查

对用户提供的数据做系统诊断：

1. **字段完整度**：每个字段的填充率，缺失哪些关键字段
2. **数据一致性**：
   - 月销量 × 均价 ≈ 月销额（允许±15%偏差）
   - Review数量与销量的比例是否合理
   - 评分是否在正常范围（3.5-5.0）
3. **异常值标注**：
   - Review数量远超同类但销量很低
   - 评分异常低（< 3.5）的产品
   - 上架时间异常
4. **数据时间窗口**：确认数据采集时间，是否为最新数据

**✅ 验证点**：输出数据质量报告，向用户汇报诊断结果，确认数据可用后进入Step 2。

#### Step 2：数据补充 + 买家决策维度发现

**数据补充：**

| 用户缺少的数据 | 补充方式 | 精度 |
|--------------|---------|------|
| Google Trends趋势 | mimo_web_search 搜索获取 | 中 |
| 采购成本 | mimo_web_search 搜索1688报价 | 中 |
| FBA配送费 | 根据产品重量尺寸用公式计算 | 高 |
| 平台佣金率 | mimo_web_search 实时查询该品类实际费率 | 高 |
| CPC广告成本 | mimo_web_search 搜索品类CPC数据 | 中 |
| 类目退货率 | 基于品类特征给出经验值 | 中 |
| 关税 | mimo_web_search 搜索该产品HS编码对应税率 | 中 |

**买家决策维度发现**：

同模式一 Step 1 的维度发现流程，但基于用户提供的完整数据，精度更高：
1. 从关键词数据中提取词根 → AI聚类 → 五维评分 → AI补全 → 人工确认
2. 基于完整数据，维度评分更准确（搜索需求和区分产品因子有真实数据支撑）

**✅ 验证点**：每项补充数据是否标注来源和置信度，维度评分是否有数据支撑。

#### Step 3：市场容量与结构分析

基于用户提供的TOP产品数据：

**市场规模计算：**
```
品类月GMV = Σ TOP_N 月销额
品类月销量 = Σ TOP_N 月销量
平均客单价 = 品类月GMV / 品类月销量
```

**品牌集中度分析：**（详见 `references/standards.md`）

**搜索趋势与销量趋势背离分析：**

| 搜索趋势 | 销量趋势 | 含义 | 决策建议 |
|---------|---------|------|----------|
| ↑ 上升 | ↑ 上升 | 真实市场扩张 | 机会信号，值得深入 |
| ↑ 上升 | → 持平或 ↓ 下降 | 供给增速超过需求 | 竞争加剧，谨慎进入 |
| ↓ 下降 | ↓ 下降 | 市场萎缩 | 高风险，不建议 |
| → 稳定 | → 稳定 | 成熟稳定市场 | 有需求但增长空间有限 |

**✅ 验证点**：GMV计算是否正确，HHI指数是否在合理范围，品牌份额是否加总≈100%。

#### Step 4：价格带深度分析 + 价格敏感度评估 + 广告压力分析

**价格带分析**：将数据按$5分段，对每个价格区间统计：
- 产品数量、总月销量、总月销额
- 平均Review数量（竞争壁垒）、平均评分（用户满意度）
- 新品数量（上架<6个月）

结合采购成本：
- 计算每个价格带的预估毛利率
- 标注"有利润的价格带"（毛利率≥30%）
- 找到"有量+有利润+壁垒适中"的最优价格交叉区间

**价格敏感度评估：**

计算价格与销量的相关系数：
- 强负相关（r < -0.6）：价格敏感度高，降价能显著提升销量
- 弱相关（-0.3 < r < 0.3）：价格敏感度低，竞争在其他维度
- 正相关（r > 0.3）：高价高销，品牌/品质驱动市场

**广告压力分析（新增）**：

对每个价格带，计算广告压力：
```
ACOS = PPC ÷ (CVR × 平均售价)
广告费占比 = ACoS × (1 - 退货率)
净利率 = 毛利率 - 广告费占比 - 其他费用占比
```

**判断标准**：
- ACoS < 毛利率的40% → 广告压力小，有推广空间
- ACoS = 毛利率的40-60% → 广告压力中等，需精细运营
- ACoS > 毛利率的60% → 广告压力大，利润被严重压缩

**价格趋势分析：**

分析过去3-6个月的价格变化：
- 稳定 → 市场健康
- 持续下降 → 价格战加剧
- 上升 → 市场升级机会

**✅ 验证点**：各价格带产品数量加总=总样本数，价格敏感度判断有数据支撑，广告压力计算正确。

#### Step 5：产品属性与销量关系 + 供需指数分析 + 场景映射

**产品属性与销量关系：**

1. **属性识别**：从产品标题、描述中提取主要属性维度
2. **属性分组**：按属性值分组统计销量表现
3. **最优配置识别**：哪个价格区间卖得最好？哪种材质/尺寸/颜色最受欢迎？

**特征归一化规范**：同模式一 Step 6，必须遵守正确合并/错误合并规则。

**供需指数分析（核心 — 新增）**：

对每个属性组合（维度组合），计算供需指数：
```
该组合Listing销量占比 = 该组合的总销量 / 全部样本总销量
该组合Listing数量占比 = 该组合的产品数 / 全部样本产品数
供需指数 = 销量占比 ÷ 数量占比
```

**正式机会判定（必须同时满足5个条件）**：

| 条件 | 标准 | 说明 |
|------|------|------|
| 样本过线 | Listing数量 ≥ 10 | 不是1-2个样本算出来的高指数 |
| 效率过线 | 供需指数 > 1 | 销量占比明显高于数量占比 |
| 平均销量 | 高于品类中位数的50% | 有真实需求支撑 |
| 产品过线 | 组合维度有明确产品意义 | 能被供应链理解和打样 |
| 卖点可表达 | 能形成卖点表达 | 能写进Listing和A+ |

**输出**：按供需指数排序的机会列表，标注每个组合是否为正式机会。

**场景-产品形态映射：**

1. 从评论、问答、搜索词中识别核心使用场景
2. 每个场景对应的产品规格建议
3. 各场景的市场容量和竞争程度

**✅ 验证点**：属性分类是否合理，供需指数计算是否正确，正式机会5条件是否严格执行。

#### Step 6：新品机会评估

按上架时间分层：
- 0-6个月（新品期）、6-12个月（成长期）、12-24个月（成熟期）、24个月以上（老品）

**判断标准：**
- 新品占比>20%且平均月销>100 → 市场对新品友好
- 新品占比<5% → 市场固化，新品难进入

**✅ 验证点**：各时间段产品数量加总=总样本数，占比加总=100%。

#### Step 7：利润精确测算

```
单件利润 = 售价 - 采购成本 - 头程运费 - 关税 - FBA配送费 - 平台佣金 - 月度仓储费 - 广告费 - 退货损耗
```

分三种情景测算：乐观（ACoS15%/退货2%）、基准（ACoS20%/退货5%）、保守（ACoS30%/退货8%）

**✅ 验证点**：成本项是否齐全，佣金率是否为该品类实际费率。

#### Step 8：综合评分 + 可进入空间

各维度评分（1-10分）：

| 维度 | 权重 | 评分依据 |
|------|------|---------|
| 市场需求 | 25% | 月销量总和、趋势判断、季节性 |
| 竞争可行性 | 25% | HHI指数、Review壁垒、新品存活率、流量可获取性 |
| 利润空间 | 20% | 基准情景净利率、单件利润、广告压力 |
| 差异化空间 | 20% | 供需指数>1的正式机会数量、属性优化空间、功能升级空间 |
| 风险可控度 | 10% | 合规、专利、供应链、季节性风险 |

**可进入空间（并行计算）**：
```
可进入空间 = 市场容量因子 × 非垄断空间因子 × 可运营性因子 × 利润余量因子
```

**✅ 验证点**：各维度评分是否有充分依据，可进入空间是否与综合评分一致。

### 输出

使用报告模板（见 `references/report-templates.md` 模式二部分），数据置信度标注为 ★★★★☆。

**HTML报告（必须）**：生成HTML格式报告，保存到 `调研报告/[品类关键词]-mode2-[日期].html`。

## 报告迭代机制

用户拿到报告后，可能需要调整假设重新测算。处理方式：

1. **用户提出调整**（如"如果售价改成$29.99呢？"）
2. **基于原数据重新计算**，不需要重新收集数据
3. **输出对比**：新结果与原结果的差异，标注变化点
4. **支持的调整参数**：售价、ACoS/退货率、采购成本、头程运费方式、目标市场份额

**✅ 验证点**：迭代计算结果与原结果对比，标注变化幅度，确认逻辑一致性。
