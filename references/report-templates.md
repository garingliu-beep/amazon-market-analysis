# Report Templates - 报告模板

> 版本：v2.2.0 | 更新日期：2026-07-07
>
> **输出格式要求**：
> - **HTML（必须输出）**：每次调研完成后，必须生成HTML格式报告并保存为 `.html` 文件。HTML报告是主要交付物。
> - **Markdown/文本**：根据目标渠道自适应，Markdown表格适用于文档/WebChat；对齐文本适用于Discord/WhatsApp。
>
> 详见下方「HTML报告输出规范」章节。

## HTML 报告输出规范（必须执行）

### 输出要求

**每次调研完成后，必须生成HTML格式报告**，具体要求：

1. **文件命名**：`[品类关键词]-[模式]-[日期].html`
   - 示例：`water-bottle-mode1-20260707.html`
   - 示例：`lunch-box-meeting-20260707.html`

2. **文件保存**：保存到工作区 `调研报告/` 目录（与SKILL文件分离）
   - 完整报告：`调研报告/[文件名].html`
   - 会议版：`调研报告/[文件名]-meeting.html`
   - 一页版：`调研报告/[文件名]-onepage.html`

3. **输出时机**：
   - 模式一调研完成 → 生成模式一HTML报告
   - 模式二调研完成 → 生成模式二HTML报告
   - 模式三调研完成 → 生成模式三HTML报告 + 会议四页版（如用户选择）
   - 快速筛选 → 生成选品一页HTML报告

4. **同时输出**：HTML文件生成后，同时在对话中以文本形式输出报告摘要（核心结论+评分+建议）

### HTML 模板文件

| 模板 | 文件路径 | 用途 |
|------|---------|------|
| 主报告模板 | `references/templates/html-report.html` | 模式一/二/三通用报告 |
| 会议四页版 | `references/templates/html-meeting.html` | 团队评审会用 |
| 选品一页版 | `references/templates/html-onepage.html` | 快速筛选用 |

### HTML 模板使用方法

1. **读取模板文件**：使用 `read` 工具读取对应模板的HTML文件
2. **替换变量**：将模板中的 `{{VARIABLE_NAME}}` 占位符替换为实际数据
3. **条件渲染**：
   - 根据模式选择性展示section（模式一不需要供需指数深度分析等）
   - 根据数据有无决定是否展示某个模块
4. **写入文件**：使用 `write` 工具将生成的HTML写入 `调研报告/` 目录
5. **通知用户**：告知用户HTML报告已生成及文件路径

### 变量命名规则

| 变量类别 | 前缀 | 示例 |
|---------|------|------|
| 综合评分 | `SCORE_` / `S1-S5` | `TOTAL_SCORE`, `S1` |
| 可进入空间 | `F1-F4`, `ENTER_SPACE` | `F1`, `ENTER_SPACE` |
| 仪表盘指标 | `D1-D9` | `D1_V`（值）, `D1_C`（颜色） |
| 供需指数 | `SD_` | `SD_OPPORTUNITY_CARDS` |
| 价格带 | `PRICE_BAND_` | `PRICE_BAND_ROWS` |
| 风险 | `RISK_` | `RISK_ROWS` |
| 颜色/等级后缀 | `_C` / `_L` | `HHI_C`（颜色class）, `HHI_L`（等级文字） |

### 颜色变量映射

| 判断结果 | CSS class | 颜色 |
|---------|-----------|------|
| 健康/绿灯/友好/优秀 | `b-g` / `si-g` | 绿色 |
| 警戒/黄灯/中等/合格 | `b-y` / `si-y` | 黄色 |
| 危险/红灯/困难/不建议 | `b-r` / `si-r` | 红色 |

### HTML 报告特性

- **响应式设计**：适配桌面和移动端
- **打印友好**：`@media print` 样式优化，支持直接打印为PDF
- **中文优化**：使用 PingFang SC + Microsoft YaHei 字体栈
- **自包含**：所有CSS内嵌，无外部依赖，可直接在浏览器中打开
- **专业配色**：深蓝渐变头部 + 白色卡片 + 状态颜色编码
